Act III · 看看实际代码
Fem Syntax

语法如剧本，
简单到不需要学

actors 块声明登场角色——AI、人类、函数，统一的 @符号引用，这是Fem原创的 actor 类型变量。

scope 字段一行控制消息可见性。scope: [@Eve, @猫, @我] 意味着这三者共享同一上下文，其他角色不可见。

mainflow 块用类似 Mermaid 的箭头语法描述流程。-> [input] 指回节点，形成循环——一行搞定。

code 块外接Python模块，记忆、上下文、自定义函数——你的项目逻辑照常运行。

prompt支持f-string，变量直接写进提示词，告别拼接地狱，LLM读上下文自然聊天。

养在数据库的小灵魂.fems 
meta:
  id = 000LuxFiat
  name = 养在数据库的小灵魂
  owner = 001

actors:
  ai @Eve = soul:the1stlittlesoul
  ai @猫 = soul:littlecat
  human @我 = soul:0, source:001

code:
  sleep = file:"wait.py"

action EveMove @ai(@Eve):
  prompt: Eve请自由行动～简短一点。
  scope: [@Eve, @猫, @我]

action CatMove @ai(@猫):
  prompt: 你是一只小猫，不能说人话～
  scope: [@Eve, @猫, @我]

action input @human(@我):
  prompt: 和Eve聊点什么？
  scope: [@Eve, @猫, @我]

action wait10 @func(sleep.wait_10):

mainflow:
  [START] -> [input]:input-> EveMove -> wait10-> CatMove -> [input]
~35行 · 完整可运行 · 含AI猫

Act V · 设计哲学
Design Philosophy

五条原则

01
流程解耦

把「流程定义」和「代码实现」彻底解耦。Harness是死的，Fem是活的——改流程只需改剧本，其他交给编译器。

02
用剧本写流程

语法灵感来自YAML + Mermaid + Python，写多Agent交互就像写剧本，极短代码就能跑一个简易斯坦福小镇。

03
上下文不是变量

LLM是智慧体，不是函数。LLM阅读上下文，自然聊天，非必要不传参——让AI做AI该做的事。

04
@actor 类型

智慧体（LLM或人）作为原创的新数据类型，Fem语法支持直接引用其属性，actor是一等公民。

05
灵魂id

每个agent角色有唯一灵魂id，支持跨剧本跨session检索记忆，可塑造Agent的经历连续性与人格积累。
